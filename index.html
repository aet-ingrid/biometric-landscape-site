The binding quality pattern in the matrix below is structurally determined by this diagram. Vertical integration across Layer 1 and Layer 2 is the decisive factor for hard credential binding in enterprise contexts — and Ping Identity is the only enterprise IAM player with this property, a direct result of the Keyless acquisition. World ID achieves the deepest binding quality (owns Layer 0 through Layer 1 end-to-end) but has no Layer 2 presence; its credentials are issued to downstream platforms, not bound within an enterprise IAM stack. Okta's soft credential binding is not a product gap that can be closed incrementally — it is structurally determined: as a Layer 2-only player, there is no layer at which Okta can anchor an identity credential to a biometric event at issuance time without acquiring or deeply integrating a Layer 1 capability.

Microsoft (P03)

Entra Agent ID produces soft binding — biometric proof from WHFB provisioning event does not travel to the agent credential; upgrade path (Face Check receipt model extended to agent credentials) architecturally understood but not shipped.

Okta (P04)

No native hard-binding mechanism anywhere in the stack; software-only token signing; soft→hard binding transition requires acquiring Nametag or executing deep third-party integration — most exposed player in enterprise IAM to regulatory pressure on this gap.

World (P05)

Token-incentivized growth model means user count claims cannot be read equivalently to SaaS adoption; issuance-side is centralized (TFH controls Orb + deduplication algorithm) despite decentralized presentation-side design.

Ping Identity (P23)

Gap between owning the Keyless ZK mechanism and having it integrated into the agent credential artifact signing chain — hard binding path is native but integration not confirmed shipped as of March 2026.
