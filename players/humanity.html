01

Key Finding

Humanity Protocol is the closest structural analog to World ID in the decentralized biometric credential lane, but lags on every decisive dimension: its primary biometric modality is palm print / geometry captured via smartphone camera (surface imaging, software liveness required) rather than iris hardware (inherent liveness assurance); its hardware root-of-trust is an optional enrollment path rather than a mandatory requirement; its ZK proof system has a public whitepaper but no confirmed independent cryptographic audit; and it has no deployed agentic AI delegation layer — absent from the strategic chokepoint where the agentic identity question will be decided. The February 2026 pivot from Proof-of-Personhood to a broader Proof of Trust credential platform, combined with the live Mastercard Open Finance integration (November 2025), represents a credible enterprise positioning move, but the reported ~9M Human ID count is severely undermined by the founder's own statement that up to 90% may be testnet bots.

03

Architecture — How the Binding Works

Human ID — ZK Proof of Trust: Binding Chain

At enrollment, a user scans their palm via either the Humanity mobile app (iOS/Android) or a dedicated Humanity hardware scanner. Palm biometric features are extracted; the system checks uniqueness against the on-chain Arbitrum registry; and an identity commitment (Human ID) is inscribed as a leaf in the registry. Biometric template data is "sharded" — distributed across nodes using a threshold scheme — so no single party retains the complete biometric.

At presentation, the user's device generates a ZK proof of registry membership, proving "I am a leaf in the registry" without revealing which leaf. The Proof of Trust layer extends this to attribute claims: age, residency, employment, income, and real-world assets can be attested via ZK proofs over web2 data retrieved via zkTLS, layered on top of the core uniqueness credential.

Why this is hard-path-native (not confirmed hard): The ZK proof architecture — if correctly implemented — would produce independently verifiable proofs against the on-chain registry without querying Humanity Protocol directly. However, the specific ZK proof scheme (proof curve, hash function, commitment parameters) is not published in the whitepaper to a level enabling independent verification. No public cryptographic audit has been confirmed. The hard binding path is architecturally claimed and structurally plausible, but cannot be independently verified from available public sources.

Two-Tier Biometric Architecture

Phase 1 — Palm print / smartphone (dominant path): Consumer smartphone camera in visible-light spectrum captures surface palm features (skin lines, loops, creases). CNN-based feature extraction. Software liveness algorithm required — the physical modality is surface imaging and cannot distinguish a living palm from a photograph by physics alone. This is the path through which the vast majority of Human IDs have been issued.

Phase 2 — Palm vein / IR hardware scanner (limited path): Dedicated Humanity hardware scanner with infrared imaging capability captures subcutaneous vein patterns. Palm vein imaging is inherently liveness-detecting because vascular patterns cannot be spoofed by surface artifacts. Hardware scanners have been deployed at events (TOKEN2049 Singapore) and university partner sites; scanner fleet is small relative to app enrollment volume.

This two-tier architecture is a meaningful distinction: users enrolled via hardware scanner have a stronger trust substrate than users enrolled via the mobile app. The aggregate binding quality at the current ~9M scale is determined by the dominant path — smartphone software enrollment — which is why the classification is hard_path_native_unaudited rather than hard.

Biometric Sharding — Distributed Custodianship Model

Humanity Protocol distributes palm scan template data across a validator node network using a threshold secret-sharing scheme, such that no single party holds sufficient data to reconstruct the original biometric. This is a structurally distinct trust substrate from both software-only (where a single server can hold complete data) and hardware root-of-trust (which addresses capture assurance, not post-capture storage). If regulators accept distributed biometric custodianship as a valid GDPR Art. 9 proportionality argument, this architecture may prove to be a regulatory differentiator vs. World ID's centralized iris data retention — which is the root cause of all 10 of World ID's active jurisdictional bans.

Arbitrum Registry

Human ID commitments are stored on Arbitrum (Ethereum L2). The on-chain registry is the presentation-time root of trust; relying parties can verify ZK proofs against the registry without querying Humanity Protocol's infrastructure. ZK proof generation is handled by dedicated zkProofer validator nodes; verification is performed by Identity Validator nodes. The zkProofer network participates in $H token staking mechanics.

Proof of Trust Extension — Attribute Credentials via zkTLS

The Proof of Trust layer (announced 2025, Mastercard integration live November 2025) extends the base uniqueness credential to verifiable attribute claims. Attributes are sourced from web2 data providers (banks, financial institutions, employment records) via zkTLS — a protocol that generates ZK proofs over HTTPS session data, attesting to data retrieved from web services without exposing the underlying session or credentials. The result is a ZK-attested claim ("this Human ID holder has verified income above X") that can be presented to relying parties without revealing the underlying financial data.

Important qualification: A zkTLS proof of income from a bank portal attests that the data appeared in an HTTPS session — it does not carry the legal equivalence of a state-issued income certificate. In regulated KYC/AML contexts, the acceptance of zkTLS-attested attribute claims as a substitute for primary-source verification is not established and remains jurisdiction-dependent.

Binding Sequence

Issuance-time (primary): The biometric-intensive event is enrollment — palm scan, feature extraction, uniqueness check, Human ID inscription on Arbitrum. Same sequence structure as World ID. The resulting Human ID is a persistent artifact.

Presentation-time: ZK proofs are generated from the Human ID commitment as needed. Attribute claims are generated via zkTLS from web2 sources on demand. No fresh biometric capture is required at presentation.

04

Strategic Position

Humanity Protocol is a protocol-company hybrid operating in the decentralized biometric credential lane, building a palm-based ZK identity system on Arbitrum that issues Human IDs using on-chain identity commitments and ZK proofs, with an attribute layer (Proof of Trust) that adds verifiable credentials for KYC, age verification, income, and real-world asset ownership via zkTLS web2 attestations.

In February 2026, Humanity Protocol pivoted its positioning from Proof-of-Personhood — confirming unique human identity in exchange for $H token rewards — to Proof of Trust: a broader biometric-anchored credential platform targeting financial services, KYC compliance, age verification, and enterprise identity. The token-reward-for-enrollment model is being wound down in favor of developer APIs and enterprise integrations. The Mastercard Open Finance partnership (live November 2025 in the US market) is the first deployed enterprise integration: Human ID holders can generate ZK credentials attesting to income, cash flow, and asset ownership using Mastercard's open banking data, without exposing underlying financial documents.

The company has raised approximately $50M in total disclosed funding ($30M April 2024 at ~$1B valuation; $20M January 2025 at $1.1B fully diluted valuation), with investors including Kingsway Capital, Animoca Brands, Pantera Capital, and Jump Crypto. The Moongate acquisition adds event and venue scanner deployment capability.

The reported ~9M Human ID count is materially unreliable: the founder has publicly stated up to 90% may be testnet bots. Humanity Protocol is not yet a scaled live-user identity network in any operationally meaningful sense. The pivot to Proof of Trust represents a strategic acknowledgment that the token-incentivized PoP enrollment model does not produce a quality user base — the enterprise credential and developer API strategy is the credible path forward.

05

Gaps & Regulatory Exposure

Critical Gap

Mobile app enrollment is dominant at current scale; software-only liveness on the smartphone path undermines the hardware-root binding quality claim. The ZK proof system's cryptographic specifics have not been independently audited. Up to 90% of reported Human IDs may be testnet bots per the founder's own statement. No agentic AI delegation layer exists.

Binding Quality Concerns

  • Software liveness dominance: At ~9M Human IDs, the vast majority were enrolled via smartphone app using software liveness detection on palm print images. This is categorically weaker than hardware-enforced liveness (World ID's Orb, or Humanity's own Phase 2 IR scanner). The aggregate binding quality of the credential pool is determined by the dominant enrollment path.
  • ZK system unaudited: The whitepaper at docs.humanity.org/whitepaper describes the architecture but does not publish cryptographic parameters (proof curve, hash function, commitment scheme). No independent cryptographic audit has been confirmed as complete and published. The brief states three audits (security, privacy, ML) are required before Phase 2 governance decentralization — their status is unconfirmed.
  • No public core code: The GitHub organization (humanity-org) has ~7 public repositories covering integration examples and smart contracts. The core biometric verification and ZK proving engine is proprietary and private — not independently auditable from public sources.

Agentic AI — Absent

Humanity Protocol has no confirmed agentic AI identity or agent delegation capability as of April 2026. World ID's AgentKit (March 2026, limited beta) positions World as the first PoP-adjacent actor to extend pseudonymous human identity into AI agent delegation chains. Humanity Protocol is absent from this chokepoint. The IETF WIMSE gap (no reference to workload identity standards for agent contexts) compounds this — if WIMSE achieves enterprise adoption, a PoP-only layer without WIMSE interoperability will be locked out of enterprise agentic AI contexts.

Regulatory Exposure

No confirmed active enforcement actions as of April 2026. Lower regulatory profile than World ID likely reflects lower deployment scale rather than a structurally cleaner architecture. The same GDPR Art. 9 biometric data processing risks apply in principle. Two potential mitigating factors vs. World ID:

  • Biometric sharding: Distributed template storage may support a stronger GDPR proportionality argument than World ID's centralized iris retention. No DPA has ruled on this architecture.
  • Mobile-app dominance: Ironically, the weaker trust substrate (smartphone enrollment) may create lower GDPR enforcement risk than dedicated hardware collection, as the consent and transparency obligations for app-based data collection are more familiar to regulators.
